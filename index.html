developer notes
(updated: may 5 15:00)

1) for editing accuracy, in the browser dev tools look at the body ID at the top of the source code for the miva code name of the actual page that is loaded (like CTGY, BASK, etc), as this is not always apparent

2) outside of a demo, if you wish to quickly click through the links below (without having to log in) do one of two things:

    a) go to customers > check box to the left of any customer > click blue button 'shop as customer' .. this should disable having to log in all the time to see a link
    b) or, go to user interface > global settings > global header & footer > global header .. and comment out lines of code 1-4, beginning with:

        <mvt:if expr="NOT g.basket:cust_id">

    this disables the need to log in.

3) the main user experience a customer will have 90% of the time is the first section below, 'shopping ux path' (demo-ready and pixel perfect for ALL 60 customers) .. the remaining pages (with a few exceptions) are account pages or category pages and such with only minor editing left to do

4) in gold links below is in-depth coding info, either access to a complete build via codepen or stackblitz, or access to my master edit log, which is a file i used to write and store all CSS editing, so if you wish to see a bunch of media queries or alignments coding, it's all there
